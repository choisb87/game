# Void Runner: 중력의 끝 — 게임 디자인 플랜

## 1. 장르 선택 근거

### 왜 중력 반전 러너인가?

| 비교 장르 | 단점 | 중력 러너 우위 |
|-----------|------|----------------|
| 무한 러너 (점프) | 점프 타이밍만 반복 → 단조로움 | 중력 반전 = 2개 표면 활용, 공간 활용도 2배 |
| 리듬 게임 | 콘텐츠(음악) 의존 | 절차적 생성으로 무한 리플레이 |
| 퍼즐 러너 | 속도감 부족 | 실시간 판단 + 속도 에스컬레이션 |
| 터치 플랫포머 | 복잡한 조작 | 원터치, 3초 안에 학습 |

**핵심**: 원터치 조작 + 즉각적 피드백 + 에스컬레이팅 텐션 = 30초 훅의 황금 공식

## 2. 코어 루프

```
탭(중력 반전) → 장애물 회피 → 크리스탈 수집 → 콤보 빌드 → 존 진입 → 속도 증가 → 더 어려운 판단 → 사망 → 즉시 재시도
```

### 30초 경험 설계

| 시점 | 이벤트 | 플레이어 감정 |
|------|--------|---------------|
| 0초 | TAP TO START, 첫 탭으로 게임 시작 | 기대감 |
| 2초 | 첫 바닥 스파이크 접근 | 긴장 → 탭! |
| 3초 | 중력 반전, 천장으로 이동 + 시안 플래시 | 쾌감, 시각 피드백 |
| 5초 | 첫 크리스탈 수집 → "+100" 텍스트 | 보상감 |
| 8초 | 두 번째 크리스탈 연속 수집 → "x2 COMBO" | 성취감 |
| 12초 | 천장 스파이크, 다시 반전 필요 | 리듬 형성 |
| 20초 | 속도 체감적 증가 | 긴장감 상승 |
| 25초 | 근접 회피 성공 | 도파민 피크 |
| 30초 | 첫 사망 → 점수 화면 → TAP TO RETRY | "한 번 더!" |

## 3. 판단 레이어 (5가지)

1. **타이밍**: 언제 중력을 반전할 것인가
2. **위치 판단**: 반전 후 어느 표면에 착지할 것인가
3. **리스크 vs 리워드**: 위험한 위치의 고가치 크리스탈을 노릴 것인가
4. **콤보 유지**: 연속 수집을 위해 위험을 감수할 것인가
5. **레이저 타이밍**: 점멸 패턴을 읽고 통과 시점 판단

## 4. 프리미엄 유료 앱 가치 제안

### 광고 없는 게임이 왜 중요한가

중력 러너는 **몰입과 리듬**이 핵심입니다:
- 전면 광고 = 리듬 파괴 = 재시도 의욕 소멸
- 배너 광고 = 화면 공간 침해 = 조작 실수 유발
- 보상형 광고 = 무과금 열등감 = 유료 유저 기만

### 유료 앱($2.99)이 제공하는 가치

| 가치 | 설명 |
|------|------|
| 완전한 게임 | 모든 기능 즉시 해금, 에너지/타이머 없음 |
| 무한 리플레이 | 절차적 생성 = 매번 다른 장애물 배치 |
| 오프라인 플레이 | 네트워크 불필요, 비행기/지하철에서 플레이 |
| 깔끔한 UX | 광고 없음, IAP 유도 없음, 방해 없음 |
| 프리미엄 비주얼 | 네온 글로우, 파티클, 화면 이펙트 |
| 원터치 마스터리 | 3초 학습, 무한 숙련 곡선 |

## 5. 리텐션 설계

### 단기 (세션 내)
- 즉시 재시도 (사망 → 1탭으로 리스타트)
- 콤보 시스템 (매번 "이번엔 더 높은 콤보를" 동기 부여)
- 존 시스템 (다음 존 도달 목표)

### 중기 (일간)
- 최고 점수 추적 (자기 기록 갱신 동기)
- 최장 거리 기록
- 존 진출 기록

### 장기 (주간+)
- 점진적 속도 증가 = 기술 성장 체감
- Zone 4+ 진입은 숙련자만 가능
- "완전 클리어" 개념 없음 → 무한 성장 여지

## 6. 기술 설계

### 아키텍처: 불변 상태 + 순수 함수

```
GameState (immutable data class)
    ↓ [update(dt)]
GameLogic (pure functions)
    ↓ [render(state)]
GameRenderer (Canvas drawing)
    ↓ [detectTapGestures]
GameScreen (Compose integration)
```

### 장점
- 상태 추적 용이 (디버깅/테스트)
- 레이스 컨디션 불가 (단일 스레드 업데이트)
- 재현성 (같은 입력 = 같은 결과)

### 절차적 생성
- 존 기반 난이도 스케일링
- 장애물 유형 풀이 존마다 확장
- 간격/크기 파라미터가 존에 따라 조정
- 크리스탈 배치: 장애물 사이 1-3개, 희귀 확률 15%

### 성능
- 60fps Canvas 렌더링
- 파티클 시스템: 수명 기반 자동 정리
- 월드 오브젝트: 화면 밖 자동 컬링
- SharedPreferences 기반 경량 영속성

## 7. 향후 확장 가능성

- 캐릭터 스킨 (다이아몬드, 삼각형, 원형 등)
- 데일리 챌린지 (시드 기반 동일 맵)
- 리더보드 (로컬/온라인)
- 신규 장애물 유형 (이동 벽, 회전 레이저, 자기장)
- 파워업 (슬로우 모션, 쉴드, 자석)
- 햅틱 피드백 (중력 반전 시 진동)
