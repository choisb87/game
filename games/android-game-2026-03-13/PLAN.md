# Flame Dash: 불꽃 질주 — 게임 디자인 플랜

## 핵심 컨셉

"아래에서 올라오는 화염을 피해 좌/우 대시로 플랫폼을 밟아 올라가는 끝없는 수직 러너"

## 30초 체험 시나리오

1. **0~3초**: 불꽃 캐릭터가 첫 플랫폼에 서있다. "TAP TO START" 표시. 아래에서 용암이 천천히 보인다.
2. **3~5초**: 탭 → 점프! 직관적으로 좌/우 탭해서 대시. 대시하면 불꽃 파티클이 뿜어진다.
3. **5~15초**: 리듬감 형성. 플랫폼이 위로 이어지고, 보석을 수집하면 콤보 텍스트가 뜬다. 바운시 플랫폼(초록)을 밟으면 슈퍼점프.
4. **15~25초**: 난이도 상승 인지. 크럼블링 플랫폼(착지 후 파괴), 아이스 플랫폼(미끄러짐) 등장. 화염 속도 증가.
5. **25~30초**: 긴장감 최고조. 화염이 바로 아래까지 오고, "⚠ DANGER" 경고. 화면 가장자리가 붉게 빛난다.

→ **첫 죽음 → "한 판만 더" 충동 → 즉시 재시작**

## 판단 레이어 (5가지 이상)

1. **좌/우 대시 선택**: 다음 플랫폼이 어디에 있는지 빠르게 판단
2. **대시 타이밍**: 쿨다운(0.3초)이 있으므로 무작정 연타 불가
3. **플랫폼 타입 판단**: 크럼블링 위에 오래 있지 말 것, 바운시는 적극 활용
4. **보석 수집 vs 안전 경로**: 보석이 위험한 곳에 있으면 리스크-리워드 판단
5. **화면 랩 활용**: 좌우 끝을 넘어서 반대편으로 이동하는 숏컷 전략
6. **낙하 중 대시**: 떨어지는 중에 대시하면 소폭 상승 — 실수 복구 수단

## 리텐션 메커니즘

### 단기 (세션 내)
- 콤보 시스템: 보석 연속 수집 → 멀티플라이어
- 화염 긴장감: 항상 아래서 올라오는 위험 → 아드레날린
- 즉시 재시작: 죽으면 한 번 탭으로 바로 다시

### 중기 (일/주)
- 하이스코어 갱신 추구
- 난이도 커브 마스터리 (플랫폼 타입별 전략 체득)
- 보석 수집 총계 누적

### 장기
- 프로시저럴 맵 → 매 플레이가 새로움
- 스킬 시링: 대시 타이밍, 화면 랩 활용 등 마스터리 곡선

## 기술 아키텍처

### 상태 관리
- `GameState`: 단일 불변 데이터 클래스
- `GameLogic`: 순수 함수로만 상태 변환 (부수효과 없음)
- `GameRenderer`: 상태를 받아 Canvas에 그리기만

### 게임 루프
- Compose `withFrameNanos` 기반 60fps
- 델타 타임 기반 업데이트 (프레임 독립적)
- 최대 dt 33ms 제한 (랙 스파이크 방지)

### 프로시저럴 생성
- 카메라 위쪽 300px 앞서서 플랫폼 미리 생성
- 난이도에 따라 플랫폼 간격/폭/타입 비율 조정
- 화염 속도: baseSpeed + acceleration * gameTime

## 향후 확장 가능성

- 일일 챌린지 (시드 기반 고정 맵)
